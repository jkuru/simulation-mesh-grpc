# example: reference-app-with-framework

**Platform golden path** — run **reference-app** on the mesh using only
**virtualization-framework** (no hand-written Istio VirtualService / EnvoyFilter).

This example is the **platform acceptance suite** (`make platform-e2e` from repo root).  
See [`docs/GOLDEN_PATH.md`](../../docs/GOLDEN_PATH.md).

| | |
| --- | --- |
| Status | [STATUS.md](./STATUS.md) |
| Framework | [`apps/virtualization-framework`](../../apps/virtualization-framework/) |
| App | [`apps/reference-app`](../../apps/reference-app/) |

## Three steps (target DX)

```bash
# 1) Platform — install framework (once per cluster)
make install-framework
# or: kubectl apply -k ../../apps/virtualization-framework/config

# 2) Service team — app workloads (mesh mode) + SimulationManifest
make apply-app
make apply-manifest
# Labels simulation.io/propagation=enabled are already on Deployments in kustomize/

# 3) Proof
make test-job
```

Full automated e2e (kind + existing Istio):

```bash
CLUSTER=servicemesh KEEP_CLUSTER=1 make e2e
```

## What is NOT in this package

| Forbidden here | Why |
| --- | --- |
| `VirtualService` / `ServiceEntry` / `EnvoyFilter` YAML | Generated by the framework from the CR |
| `apps/reference-app/kube/overlays/dev` teaching overlay | That path is for learning mesh by hand (v1) |

Check:

```bash
make assert-no-istio-in-kustomize
```

## Prerequisites

- Docker, kind, kubectl  
- Istio installed on the cluster  
- Images: built by `e2e` script (`reference-app/*`, `virtualization-framework`)  

## Notes

This package is **YAML + scripts** (not a Go module). Quality is enforced by:

- `make assert-no-istio-in-kustomize` — no hand-written VS/SE/EnvoyFilter  
- `make e2e` — full cluster proof
